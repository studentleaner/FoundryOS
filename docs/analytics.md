# Product Analytics

Data is the foundation of the Decision Engine. The Analytics layer is designed to measure behavior, not opinions—and to tie every metric to a decision.

## Frameworks

### 1. AARRR Funnel (Pirate Metrics)
The Startup OS evaluates traction through the AARRR funnel:
- **Acquisition:** Where do users come from?
- **Activation:** Did they experience the core value?
- **Retention:** Did they return?
- **Referral:** Are they telling others?
- **Revenue:** Is there a transaction?

### 2. North Star Metric
The singular metric that defines value creation for the user and capture for the business. This varies by product, but there should only ever be one active North Star Metric during a growth phase.

### 3. Event-Based Tracking
We do not track "page views". We track actions.
- Action happening → Event captured → Insight generated → Change made.

## Deliverables
- **Event Taxonomy:** A centralized document defining every event the application fires, including core properties.
- **Dashboards:** Visual representations of the AARRR funnel and cohorts.
- **Weekly Review Reports:** Automated summaries of metric movements.

## Kill Signals
- If the metrics are implemented but ignored by the founders.
- If data contradicts opinions, but opinions dictate the roadmap.
