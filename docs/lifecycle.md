# The Complete Startup Lifecycle

The Startup OS forces every project through an irreversible, evidence-backed lifecycle. Progress is gated by real-world validation, not feelings.

## Stage 0: Ideation & Intent
- **Goal:** Avoid building "interesting nonsense." Anchor on real pain.
- **Questions:** Who hurts? How often? What do they do today?
- **Deliverables:** One-line problem statement, ICP definition, Pain score.
- **Kill Signal:** The problem is a "nice to have" and lacks urgency.

## Stage 1: Validation by Behavior
- **Goal:** Validate behavior, not opinions. Find the execution signal.
- **Questions:** Do users acknowledge the pain? Will they switch?
- **Deliverables:** Validation score, Evidence log (payments, emails), Pivot/Proceed decision.
- **Kill Signal:** "Sounds cool" feedback without any behavioral change.

## Stage 2: Business Design & Differentiation
- **Goal:** Design a defensible execution model based on competitor gaps.
- **Questions:** Where do users struggle today? What is manual or broken?
- **Deliverables:** Execution gap analysis, Unit economics hypothesis (LTV:CAC).
- **Kill Signal:** The idea is only linearly better (10% better) with no meaningful wedge.

## Stage 3: MVP Design & Architecture
- **Goal:** Define the minimal system required for value delivery.
- **Questions:** What must exist? What can be brutally removed?
- **Deliverables:** MVP definition, Architecture diagram, Hosting plan.
- **Kill Signal:** Uncontrolled scope creep; trying to appeal to too many personas.

## Stage 4: Execution & Build
- **Goal:** Ship the minimal product and test assumptions.
- **Questions:** Does the pipeline work? Can it handle failure?
- **Deliverables:** Live URL, Deployment status, Error baseline.
- **Kill Signal:** System cannot reliably deliver the core journey.

## Stage 5: Observability & Product Analytics
- **Goal:** Measure reality using AARRR (Acquisition, Activation, Retention, Referral, Revenue).
- **Questions:** Where do users drop? What is the North Star metric?
- **Deliverables:** Analytics dashboard, Event taxonomy, Alert rules.
- **Kill Signal:** No one uses the data to make decisions.

## Stage 6: Go-To-Market (GTM)
- **Goal:** Create repeatable, measurable acquisition channels.
- **Questions:** Where are the users? What message triggers action?
- **Deliverables:** Single-channel GTM plan, SEO keyword map, Campaign metrics.
- **Kill Signal:** Spreading across multiple channels with no measurable CAC.

## Stage 7: Feedback, Iteration & PMF
- **Goal:** Achieve Product-Market Fit (PMF) and fix retention.
- **Questions:** Why do users stay or leave?
- **Deliverables:** Retention cohorts, Feedback inbox, Product update log.
- **Kill Signal:** Artificial growth with massive user churn.

## Stage 8: Decision Engine (Scale, Pivot, or Kill)
- **Goal:** Decide the fate of the project based on the collected evidence.
- **Questions:** Does this deserve the next 90 days of life?
- **Deliverables:** Scale roadmap, Exit plan, Valuation logic.
- **Decision Matrix:**
  - **Double-down:** If unit economics are positive and retention is stable.
  - **Pivot:** If the problem exists but the solution failed the behavioral test.
  - **Kill:** If assumptions are invalidated cleanly.
